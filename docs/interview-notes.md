# 面试讲法

## 30 秒项目介绍

我做了一个电商营销自动化 Agent Demo，目标是把传统营销后台里的活动查询、渠道归因、人群圈选、优惠券策略和 ROI 诊断能力，封装成 Agent 可调用的业务工具。运营可以用自然语言提问，系统先做 RAG 检索，再调用后端工具获取数据，最后给出诊断结论。对批量发券这类高风险动作，Agent 只生成待审批动作，不直接执行。

## 面试官可能追问

### 为什么不用一个普通 Chatbot？

普通 Chatbot 只能回答文本，无法可靠访问业务数据。Agent 的核心是工具调用和工作流：它能先查活动配置，再查渠道漏斗，再结合知识库生成结论。

### RAG 解决什么问题？

RAG 用来提供业务上下文，比如营销 SOP、归因口径、活动规则、风控规范。没有 RAG，模型容易凭经验编答案；有 RAG 后可以基于企业内部知识回答。

### 工具调用最大的风险是什么？

最大风险是越权、误调用和不可追溯。所以工具要有 schema、权限、参数校验、幂等、审计日志。写操作要经过人工确认。

### 你这个 Java 后端经验怎么迁移到 Agent？

Agent 落地需要大量后端工程能力：接口治理、异步任务、幂等、补偿、限流、权限、审计、监控和稳定性。这些正是传统 Java 后端的优势。

### 生产化还差什么？

需要接入真实 LLM、向量库、统一权限系统、真实业务工具、评测集、链路追踪、成本监控、Prompt Injection 防护和灰度发布。
